# spanpaper v0.4 — plan

Two threads of work on this branch:

1. **Presets** (the new user-facing feature) — named, saved
   configurations the user can switch between with one click / one
   command, plus cycle-next / cycle-prev for hotkey-friendly browsing.
2. **Architectural cleanup** from the v0.3.1 retrospective —
   shared-lib split, tests, CI, daemon-side thumbnails, config-migration
   warnings, `install` → `enable` rename, Budgie FIXME.

Sequencing: presets first (highest user value, mostly additive). The
architectural items pull double-duty on the presets work — the
lib-split in particular makes the tray's preset-list rendering cleaner
than re-parsing TOML inline. So: lib-split early in the branch, then
presets, then the smaller polish items, then ship as v0.4.0.

## A. Presets

### Data model

Inline in `~/.config/spanpaper/config.toml`, not a separate file —
one config file stays the simplest mental model and the existing
atomic-write + SIGHUP plumbing covers it for free.

```toml
# Active state (what the daemon renders right now).
span = "/path/to/active-span.mp4"
side = "/path/to/active-side.jpg"
audio = false
span_outputs = ["HDMI-A-4", "DP-6"]
side_output = "DP-5"
span_direction = "vertical"
span_fit = "crop"
side_fit = "crop"

# Name of the preset that the active state matches, if any.
# Set by `spanpaper preset load NAME` and `spanpaper preset save NAME`;
# cleared (or kept stale) if the user runs `spanpaper set` to change a
# field that diverges from the matching preset.
active_preset = "spicy-grok"

# Saved presets. Order matters for next/prev cycling — newest at the
# end, like a history. Names are arbitrary strings (kebab-case
# encouraged; CLI rejects whitespace and slashes).
[[presets]]
name = "spicy-grok"
span = "/home/ianm/Videos/Grok/Spicy/grok-...-mp4"
side = "/home/ianm/Videos/Grok/Spicy/grok-...-mp4"
audio = false
span_fit = "crop"
side_fit = "fit"
span_direction = "vertical"

[[presets]]
name = "nature-still"
span = "/home/ianm/Pictures/forest-1920x2160.jpg"
side = "/home/ianm/Pictures/lake.jpg"
audio = false
span_fit = "fit"
side_fit = "fit"
span_direction = "vertical"
```

Note: `span_outputs` and `side_output` are NOT part of a preset —
they're hardware identity (which monitor is which). A preset
recorded on Rig A shouldn't try to dictate output names on Rig B.

### CLI

```bash
spanpaper preset save NAME            # snapshot current Config fields into a named preset.
                                      # overwrites if NAME exists. sets active_preset = NAME.
spanpaper preset list                 # print names; mark the active one with "*".
spanpaper preset load NAME            # copy preset fields into active Config + SIGHUP daemon.
                                      # sets active_preset = NAME.
spanpaper preset delete NAME          # remove from the list. clears active_preset if it matched.
spanpaper preset rename OLD NEW       # rename in-place. updates active_preset if it matched.
spanpaper preset next                 # find active_preset's position, advance one, load it.
                                      # wraps from last back to first.
spanpaper preset prev                 # ditto, opposite direction.
```

Reuse existing path-validation guard for `save` (rejects names with
`\n`, `\0`, `/`, leading `.`, empty).

`next` / `prev` semantics when nothing is active or the active name
no longer exists: load index 0 (cycle entered cleanly).

### Tray menu

Replace the current top-of-menu structure with a Presets section
when at least one preset exists:

```
Open palette                          (existing)
─────────────
Presets ▶
  ✓ spicy-grok          ← currently active gets a checkmark
    nature-still
    ─────────
    Save current as…    ← opens a GTK text-entry dialog
    Next preset
    Previous preset
─────────────
Pause playback                        (existing)
… etc …
```

"Save current as…" pops a small dialog with a single `GtkEntry` and
OK/Cancel. Enter triggers OK. Name validation matches the CLI guard.

When no presets are saved, the submenu shrinks to just **Save
current as…** so the entry point is always reachable.

### Cycle-next / cycle-prev hotkeys

The tray menu exposes Next/Previous items, but the real value of
cycle is keyboard shortcuts bound at the compositor level (Sway,
Hyprland, Budgie keybindings). Document the recipe in the README so
users can wire `Super+Mod1+Right` → `spanpaper preset next`.

### Side effects of `spanpaper set`

When the user runs `spanpaper set --span …`, the active config
diverges from `active_preset`. Two options:

* **Strict**: clear `active_preset` to `None` (no preset is active
  anymore). Pros: honest. Cons: cycle-next from a tweaked preset
  jumps to index 0 instead of the next preset.
* **Loose**: keep `active_preset` even when it doesn't match. Cons:
  the tray's checkmark lies. Pros: cycle works from where the user
  last loaded.

Go with **strict + warning**: clear `active_preset` and `tracing::info!`
that the user diverged. Cycle behavior is predictable; the tray's
checkmark shows truth.

## B. Architectural cleanup

In rough order of impact. None of these affect users directly; all
make the next round of work cheaper.

1. **Lib + two bins split.** Move `config`, `ipc`, `media`,
   `outputs`, and a thin `daemon::pid_file()` helper into
   `src/lib.rs`. The tray binary stops re-implementing
   `daemon_alive()`, the IPC client, `PartialConfig`. Estimated:
   ~200-line diff, mostly moves.

2. **Daemon-side thumbnail pre-computation.** When `spanpaper set
   --span … --side …` rewrites config, also call
   `thumbnail::ensure(path)` for any changed slot. The tray's
   palette goes from "async spinner → swap in picture" to "load
   PNG, done." Removes the `gio::spawn_blocking` + weak-ref +
   suppress-autoclose code path from `palette.rs`. Pure win.

3. **Tests.**
   * `config::Config` TOML round-trips with unknown / deprecated
     fields (catches future `side_mode`-style regressions).
   * `workers::plan` for the 6 image/video × span/side
     permutations — assert which `WorkerKind` variant comes back.
   * `workers::side_swaybg_mode` translation table.
   * `cli::validated_media_path` rejects newlines / NULs, accepts
     normal paths.
   * Round-trip of `[[presets]]` config: save then load.

4. **CI.** `.github/workflows/ci.yml` runs `cargo fmt --check`,
   `cargo clippy -- -D warnings`, `cargo test`, and `cargo build
   --features tray` on every push and PR. ~30 lines of YAML.

5. **Config migration warnings.** When `Config::load` parses an
   unknown / deprecated field, emit a `tracing::warn!` naming the
   field. (Toml's serde already silently drops them; we need to
   detect them via a separate strict-parse step that lists what was
   dropped.) Catches the silent `side_mode` migration story.

6. **Rename `spanpaper install` → `spanpaper enable`.** Keep
   `install` as a hidden alias for back-compat. Reason: `install`
   reads as "install the package" when the package is already
   installed. `enable` describes what it does (enable autostart +
   optionally start now).

7. **Budgie FIXME with upstream-issue link.** The structural-diff
   menu workaround in `src/bin/spanpaper-tray/main.rs:107-220` is a
   real hack. File the upstream Budgie bug, link from a `// FIXME:`
   comment.

## Shipping

v0.4.0 ships **A. Presets** + **B.1-B.4** (lib split, daemon-side
thumbnails, tests, CI). B.5-B.7 are quality-of-life and can land
during the cycle or get deferred to v0.4.1.

The first commit on this branch is this plan doc; subsequent commits
implement the items. v0.4.0 cut via `./release.sh 0.4.0 -y` whenever
A is shippable and B has at least the lib split + tests.
